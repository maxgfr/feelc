# Architecture Decision Records

The detailed records of why feelc is built the way it is. For a one-line summary of all of them, see
[`docs/decisions.md`](../decisions.md) (also on the [docs site](https://maxgfr.github.io/feelc/docs/decisions.html)).

**Lifecycle — append-only.** An accepted ADR's Decision is never rewritten: a non-reversing follow-up is a
dated `## Update` on the same ADR; a reversal is a new *superseding* ADR (and the old one's Status flips to
`Superseded by ADR NNNN`). Copy [`0000-template.md`](0000-template.md) for a new one (number = highest + 1).
